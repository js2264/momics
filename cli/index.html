momics¶

Command-line software to manage momics repositories.

momics [OPTIONS] COMMAND [ARGS]...

Check out our docs at https://js2264.github.io/momics/ for more details

binnify¶

Binnify chromosomes from a Momics repository.

momics binnify [OPTIONS] MOMICS_REPO

Options

-w, --width <width>¶

Required The width of each bin.

-s, --step <step>¶

Required The step size for tiling.

-c, --cut_last_bin_out¶

Remove the last bin in each chromosome, which likely does not have the same width.

Default:

False

-o, --output <output>¶

Path to a bed file to write

Arguments

MOMICS_REPO¶

Required argument

Path to a momics repository

consolidate¶

Consolidate a momics repository.

Consolidation is the process of compacting all the fragments from the repository to remove any unused space. This process is useful to reduce the size of the repository and improve performance.

The vacuum flag indicates wether the consolidated array should be vacuumed after consolidation. This process is useful to further reduce the size of the repository.

momics consolidate [OPTIONS] MOMICS_REPO

Options

-v, --vacuum¶

Flag to indicate wether fragments should be vacuumed after consolidation.

Arguments

MOMICS_REPO¶

Required argument

Path to a momics repository

Copy sequence/track/feature set from a momics repo to a fa/bigwig/bed file.

-l, --label <label>¶

For track and features types, the name of the track or feature set to extract.

-o, --output <output>¶

Required Path of output file to write.

-f, --force¶

Force overwrite of existing files.

Default:

False

MOMICS_REPO¶

Required argument

Path to a momics repository

ingest¶

Ingest a data file to a Momics.

momics ingest [OPTIONS] COMMAND [ARGS]...
chroms¶

Register chromosomes sizes to Momics.

momics ingest chroms [OPTIONS] MOMICS_REPO

Options

-f, --file <file>¶

Required UCSC-style coordinates

-g, --genome <genome>¶

Genome reference (e.g. hg38, sacCer3, …).

Arguments

MOMICS_REPO¶

Required argument

Path to a momics repository

features¶

Ingest genomic features to Momics.

momics ingest features [OPTIONS] MOMICS_REPO

Options

-f, --file <file>¶

Required Named BED file, provided as –file key=value (e.g. –file bw1=my_file.bw). The –file option can be provided several times. The first three columns of the BED file must describe the genomic coordinates of the features (chromosome, start, end).

-@, --threads <threads>¶

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO¶

Required argument

Path to a momics repository

seq¶

Ingest genomic sequence to Momics.

momics ingest seq [OPTIONS] MOMICS_REPO

Options

-f, --file <file>¶

Required Fasta file

-@, --threads <threads>¶

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO¶

Required argument

-@, --threads <threads>¶

query¶

Query a Momics repository.

momics query [OPTIONS] COMMAND [ARGS]...
seq¶

Extract chromosomal sequences over chromosome intervals.

momics query seq [OPTIONS] MOMICS_REPO

Options

-c, --coordinates <coordinates>¶

UCSC-style coordinates

-f, --file <file>¶

BED file listing coordinates to query. If provided, coordinates is ignored.

-o, --output <output>¶

Output file to save data in (data will be exported as fasta)

-@, --threads <threads>¶

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO¶

Required argument

Path to a momics repository
