CLI Reference

momics

Command-line software to manage momics repositories.

momics [OPTIONS] COMMAND [ARGS]...

Options

-v, --version

Show the version and exit.

Check out our docs at https://js2264.github.io/momics/ for more details

binnify

Binnify chromosomes from a Momics repository.

momics binnify [OPTIONS] MOMICS_REPO

Options

-w, --width <width>

Required The width of each bin.

-s, --step <step>

Required The step size for tiling.

-c, --cut_last_bin_out

Remove the last bin in each chromosome, which likely does not have the same width.

Default:

False

-o, --output <output>

Path to a bed file to write

Arguments

MOMICS_REPO

Required argument

cp

Copy sequence/track/feature set from a momics repo to a fa/bigwig/bed file.

momics cp [OPTIONS] MOMICS_REPO

Options

-t, --type <type>

Required Type of data to extract

Options:

sequence | track | features

-l, --label <label>

For track and features types, the name of the track or feature set to extract.

-o, --output <output>

Required Path of output file to write.

-f, --force

Force overwrite of existing files.

Default:

False

Arguments

MOMICS_REPO

Required argument

create

Initiate a Momics repository.

MOMICS_REPO

Required argument

delete

Delete a momics repo.

MOMICS_REPO

Required argument

ingest

Ingest a data file to a Momics.

momics ingest [OPTIONS] COMMAND [ARGS]...

chroms

Register chromosomes sizes to Momics.

momics ingest chroms [OPTIONS] MOMICS_REPO

Options

-f, --file <file>

Required UCSC-style coordinates

-g, --genome <genome>

Genome reference (e.g. hg38, sacCer3, …).

Arguments

MOMICS_REPO

Required argument

features

Ingest genomic features to Momics.

momics ingest features [OPTIONS] MOMICS_REPO

Options

-f, --file <file>

Required Named BED file, provided as –file key=value (e.g. –file bw1=my_file.bw). The –file option can be provided several times. The first three columns of the BED file must describe the genomic coordinates of the features (chromosome, start, end).

-@, --threads <threads>

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO

Required argument

seq

Ingest genomic sequence to Momics.

momics ingest seq [OPTIONS] MOMICS_REPO

Options

-f, --file <file>

Required Fasta file

-@, --threads <threads>

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO

Required argument

tracks

Ingest tracks to Momics.

momics ingest tracks [OPTIONS] MOMICS_REPO

Options

-f, --file <file>

Required Named track file, provided as –file key=value (e.g. –file bw1=my_file.bw). The –file option can be provided several times.

-@, --threads <threads>

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO

Required argument

ls

List tracks/chromosomes/features registered in a Momics.

momics ls [OPTIONS] MOMICS_REPO

Options

-t, --table <table>

Which supporting table to list.

Default:

'tracks'

Options:

tracks | chroms | features

Arguments

MOMICS_REPO

Required argument

query

Query a Momics table

momics query [OPTIONS] COMMAND [ARGS]...

seq

Extract chromosomal sequences over chromosome intervals.

momics query seq [OPTIONS] MOMICS_REPO

Options

-c, --coordinates <coordinates>

UCSC-style coordinates

-f, --file <file>

BED file listing coordinates to query. If provided, coordinates is ignored.

-o, --output <output>

Output file to save data in (data will be exported as fasta)

-@, --threads <threads>

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO

Required argument

tracks

Extract track coverages over a chromosome interval.

momics query tracks [OPTIONS] MOMICS_REPO

Options

-c, --coordinates <coordinates>

UCSC-style coordinates

-f, --file <file>

BED file listing coordinates to query. If provided, coordinates is ignored.

-o, --output <output>

Output file to save data in (data will be exported as tsv)

-@, --threads <threads>

Number of threads to use in parallel operations (default: 1)

Arguments

MOMICS_REPO

Required argument
